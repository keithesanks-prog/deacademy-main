Nested Basics: The Backpack

Rob: "So you mastered the sticky note? Good. Now, what if the data isn't just flat? What if the Object has another Object inside it?"

Think of an Address inside a Customer Profile.
Do we need to flatten it? NO! It's just one backpack with a front pocket. We just need to reach deeper.
üéí
The Main Bag

Top Level Data

Things like Name and Age are loose in the main bag. Just grab them.

DATA:name

üëõ
The Front Pocket

Nested Objects

For City, you have to open the Bag, then the Pocket.

DATA:address.city

1. Create the Environment

Let's make a fresh table for this experiment.

CREATE OR REPLACE DATABASE JSON_DB;
USE SCHEMA PUBLIC;
CREATE OR REPLACE TABLE JSON_TBL1 (DATA VARIANT);
2. Insert Nested Data

Notice the address object inside the main object. It contains street, city, and zip.

INSERT INTO JSON_TBL1 (data)
SELECT PARSE_JSON('{
  "name": "Jane Doe",
  "age": 25,
  "isStudent": true,
  "address": {
    "street": "456 Oak St",
    "city": "Somewhere",
    "zip": "67898"
  }
}');
3. The Reach (Dot Notation)

This is the key lesson. We use the Colon : to enter the Variant, and the Dot . to go down a level.

SELECT
  -- Easy reach (Top Level)
  data:name::STRING AS full_name,

  -- Deep reach (Into the Pocket)
  data:address.street::STRING AS street,
  data:address.city::STRING AS city,
  data:address.zip::STRING AS zip
FROM JSON_TBL1;
üí° Rob's Rule: If it's just one object inside (like one address), use the Dot. If it's a list of objects (like multiple addresses), use LATERAL FLATTEN.
