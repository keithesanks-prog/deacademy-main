Semi-Structured Data

Project: Nested Sales Data ("The Matryoshka Problem")

Rob: "Wait! This JSON file looks different. It's not just a box of items (Array)... it's a Dictionary of items!"

In sales_data.json, the companies aren't in a list []. They are Keys in an object {}.
This means we can't just list the values. We need to extract the names from the keys themselves. And it goes deep.

Data is the Key

"Tech Supplies Inc." is not a value, it's the KEY.

We use .key to get the name.

Let's learn how to flatten this nested structure step-by-step.

1
Environment Setup

Standard procedure. We need a Database, Schema, and a Stage.

CREATE DATABASE IF NOT EXISTS SALES_PROJECT; USE DATABASE SALES_PROJECT; CREATE SCHEMA IF NOT EXISTS RAW_DATA; -- Create the internal stage for upload CREATE OR REPLACE STAGE SALES_STAGE;
2
Ingest the JSON
üëÜ UI ACTION REQUIRED:
  1. Navigate to Data » Databases » SALES_PROJECT » RAW_DATA » Stages.
  2. Select SALES_STAGE.
  3. Click + Files button and upload sales_data.json.

Create the Variant Table and Load the file.

CREATE OR REPLACE TABLE RAW_SALES_JSON ( JSON_DATA VARIANT ); COPY INTO RAW_SALES_JSON FROM @SALES_STAGE/sales_data.json FILE_FORMAT = (TYPE = 'JSON' STRIP_OUTER_ARRAY = FALSE);
3
The Triple Flatten

This is where the magic happens. We need to go 3 levels deep.

The Logic:

  • Level 1 (Companies): Flatten json_data:companies » Result alias: comp
  • Level 2 (Regions): Flatten what was inside comp » comp.value:regions » Result alias: reg
  • Level 3 (Products): Flatten what was inside reg » reg.value:products » Result alias: prod
SELECT -- The Key IS the Name (e.g. "Tech Supplies Inc.") comp.key::STRING AS COMPANY_NAME, -- The Nested Key is the Region (e.g. "North America") reg.key::STRING AS REGION, -- The Inner Key is the Product (e.g. "Laptop") prod.key::STRING AS PRODUCT_NAME, -- Finally, the values inside the product object prod.value:revenue::FLOAT AS REVENUE, prod.value:units_sold::INT AS UNITS_SOLD FROM RAW_SALES_JSON, LATERAL FLATTEN(input => json_data:companies) comp, -- 1. Unpack Company LATERAL FLATTEN(input => comp.value:regions) reg, -- 2. Unpack Region LATERAL FLATTEN(input => reg.value:products) prod; -- 3. Unpack Product
4
Create the Master View

Let's save this logic so we can query it anytime easily. This turns our semi-structured mess into a beautiful, analyzable table.

CREATE OR REPLACE VIEW SALES_DATA_DETAILS_VIEW AS SELECT comp.key::STRING AS COMPANY_NAME, reg.key::STRING AS REGION, prod.key::STRING AS PRODUCT_NAME, prod.value:revenue::FLOAT AS REVENUE, prod.value:units_sold::INT AS UNITS_SOLD, -- Bonus: Grab Sales Period from the top level company object comp.value:sales_period::STRING AS SALES_PERIOD FROM RAW_SALES_JSON, LATERAL FLATTEN(input => json_data:companies) comp, LATERAL FLATTEN(input => comp.value:regions) reg, LATERAL FLATTEN(input => reg.value:products) prod;

‚úÖ Now you can run:

SELECT * FROM SALES_DATA_DETAILS_VIEW WHERE REVENUE > 100000;
