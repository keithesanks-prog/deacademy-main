JSON Basics: The Sticky Note

Rob: "Before we unpack big shipping containers (Arrays) or Matryoshka dolls (Nested Objects), let's look at a simple **Sticky Note**."

This is Flat JSON. It has no hierarchy. It's just a simple list of properties on a single object.
For this, we don't need a heavy machine like LATERAL FLATTEN. We just need to point at what we want.

Because it's flat, we just use the Colon : to grab the value. Boom. Done.

1. Create the Board (Table)

We start fresh. Simple Variant table.

CREATE OR REPLACE DATABASE JSON_DB;
CREATE OR REPLACE TABLE JSON_TBL (DATA VARIANT);
2. Write the Note (Insert)

We use PARSE_JSON to turn a text string into a real Object.

INSERT INTO JSON_TBL (DATA)
SELECT PARSE_JSON('{
  "name": "Alice",
  "age": 30,
  "city": "New York"
}');
3. Flattening (Without Hierarchy)

Here is the trick. Since there is no list, we don't "Flatten" rows. We "Flatten" the structure into columns.

SELECT
  DATA:name::STRING as Name,
  DATA:age::INT as Age,
  DATA:city::STRING as City
FROM JSON_TBL;
