Your Cheat Sheet for Cloud, Data & Code

Stage Snowflake
A temporary storage location where data files sit before being loaded into tables.
Analogy: The "Loading Dock" or "Airport Terminal" where cargo waits.
Warehouse Snowflake
A cluster of compute resources (CPU/RAM) used to execute queries and load data.
Analogy: The "Muscle" or "Construction Crew". It does the heavy lifting.
Snowpipe Snowflake
A serverless service that automatically detects and loads files as soon as they arrive in a stage.
Analogy: A "Conveyor Belt" that stays on 24/7.
Time Travel Snowflake
A feature allowing you to query data as it existed in the past (up to 90 days).
Analogy: A "DVR / Replay Button" for your database history.
Zero-Copy Clone Snowflake
Creating a duplicate of a database/table instantly without copying the physical data storage.
Analogy: "Sharing a Google Doc" link instead of photocopying the paper.
File Format Snowflake
A set of rules (delimiter, skip header) for loading data. Can be In-line or Named.
Analogy: Coffee Order. In-Line = "Complex Order". Named = "The Usual".
Batch Loading Snowflake
Loading large amounts of data at scheduled intervals (e.g., Nightly). Uses `COPY INTO`.
Analogy: "The Moving Truck". Efficient, but high latency.
Continuous Loading Snowflake
Loading data instantly as soon as files arrive in a stage. Uses Snowpipe.
Analogy: "The Conveyor Belt". Low latency, always running.
S3 AWS
Simple Storage Service. Object storage for files, backups, and data lakes.
Analogy: "Infinite Google Drive" for applications.
Lambda AWS
Serverless compute that runs code in response to events without managing servers.
Analogy: An "On-Demand Taxi" - you pay only for the trip, not for owning the car.
SQS AWS
Simple Queue Service. Decouples systems by holding messages until a consumer is ready.
Analogy: The "Waiting Room" or "Deli Ticket Counter".
SNS AWS
Simple Notification Service. Sends messages to many subscribers (Fan-out).
Analogy: A "Megaphone" or "Group Text".
CAP Theorem Design
In a distributed system, you can only pick 2: Consistency, Availability, Partition Tolerance.
Analogy: The "Burger Franchise". If the phone line cuts (P), you either close (C) or guess the price (A).
ACID Design
Atomicity, Consistency, Isolation, Durability. Guarantees for banking-style transactions.
Analogy: "All or Nothing". Like buying a vending machine snack - money leaves AND snack falls, or neither happens.
Normalization Design
Organizing data to minimize redundancy (3NF). Good for writes (OLTP).
Analogy: "Marie Kondo" for data. Everything has one specific place.
List Python
Ordered, mutable collection of items. Defined by `[]`.
Analogy: A "Backpack" where you can throw in anything and change it later.
Dictionary Python
Key-Value pairs. Unordered (pre-3.7). Defined by `{}`. Fast lookups.
Analogy: A "Phonebook" or "Physical Dictionary". Look up a word (Key) to get meaning (Value).
DataFrame Python
2D labeled data structure with columns of potentially different types (Pandas).
Analogy: A "Programmable Excel Sheet".
