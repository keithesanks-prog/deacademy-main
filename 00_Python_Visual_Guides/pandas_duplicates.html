A Simple Guide to Pandas Duplicates

üçæ The Party Scenario

Imagine you are checking people into a party. Alice signs up. Then Bob signs up. Then... Alice signs up again by mistake?

We can't give Alice two gift bags! That's a duplicate.

1. Alice Kept
2. Bob Kept
3. Alice Duplicate!
4. Charlie Kept

In Pandas, a row is a duplicate if every single column matches another row.

Who is on the list twice?

Use .duplicated() to ask: "Have I seen this person before?"

# Check the guest list print(df.duplicated())
0 False (First time seeing Alice) 1 False (First time seeing Bob) 2 True (WAIT! We saw Alice already!) dtype: bool

üí° Quick Count

How many extra people are there?

df.duplicated().sum()

Clean up the List

Use .drop_duplicates() to kick out the extra copies.

‚úÖ drop_duplicates()

üìê Anatomy of the Command

üßπ

Think of it like a Vacuum Cleaner:

drop_duplicates() is the tool (the vacuum).

subset=['Name'] is the nozzle setting (focus only on names).

Term In Code Description
Method drop_duplicates() The action (Remove).
Parameter subset= The setting knob.
Argument ['Name'] The value you choose for the setting.
Parameter keep= Start or End preference.
Argument 'last' Keep the final one.

"I am setting the subset parameter to the argument ['Name']."

# The "One-Click" Fix df_clean = df.drop_duplicates()
ID Name 0 1 Alice <-- Kept (First one) 1 2 Bob 3 3 Charlie # The second 'Alice' is gone. Perfect!

Advanced Options (Optional)

Sometimes the rules are tricky:

  • "Last one is correct": Maybe Alice fixed a typo in the second entry?
    df.drop_duplicates(keep='last')
  • "Check Name Only": Maybe their ID changed but Name is same?
    df.drop_duplicates(subset=['Name'])
