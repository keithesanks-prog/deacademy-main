Think in 3D: Schema, Vertical Slices, and Horizontal Slices

The DataFrame as a 3D Structured Object

Stop thinking of it as text. Think of it as a physical object with structure.

üü¶ Front Face (Schema)

The header, index, and column names. This is the "Identity" of the data.

üìô Vertical Slices (Columns)

Containers of ONE type of data (e.g., all Ages). Changing type changes the substance inside.

üìó Horizontal Slices (Rows)

Single records. Adding a row stacks a new slice on the bottom.

‚úèÔ∏è 1. Changing the Front Face (Schema)

Operation: Renaming a Column.

The Code:
# Change label on the box df.rename(columns={'col 1': 'Full Name'}, inplace=True)

The text on the "col 1" box on the top surface is erased and replaced with "Full Name".

üìù 2. Changing a Vertical Slice (Type)

Operation: Type Conversion.

The Code:
# Convert Text to Number df['Age'] = df['Age'].astype(int)

The physical substance inside the 'Age' column slice changes from "Text Clay" to "Number Metal".

‚ûï 3. Adding a Vertical Slice (New Column)

Operation: Adding a Column.

The Code:
# Add new slice df['Country'] = 'USA'

A brand new vertical slice is glued to the right side of the cube. It is filled with 'USA'.

‚ûï 4. Adding a Horizontal Slice (New Row)

Operation: Concatenation / Adding Record.

The Code:
# Create Slice new_row = pd.DataFrame([['Eve', 45]], columns=['Name', 'Age']) # Stack it on bottom df = pd.concat([df, new_row], ignore_index=True)

The cube is extended downward. A new horizontal slice is stacked at the bottom (Index increases).
