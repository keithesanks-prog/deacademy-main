üîó How the Connections Work

Primary Keys (PK)

  • Unique identifier for each row
  • Cannot be NULL or duplicate
  • Example: product_key = 123
  • Think: Social Security Number

Foreign Keys (FK)

  • References a primary key in another table
  • Creates the relationship/connection
  • Example: fact_sales.product_key ‚Üí dim_product.product_key
  • Think: Employee badge linking to employee record
üí° Key Concepts

Fact Table (Center)

  • What: Measurements & transactions
  • Contains: Foreign keys + metrics
  • Example: Sales, orders, clicks
  • Size: Millions/billions of rows
  • Think: The receipt

Dimension Tables (Points)

  • What: Context & descriptions
  • Contains: Descriptive attributes
  • Example: Products, customers, dates
  • Size: Thousands/millions of rows
  • Think: The details (who, what, when, where)
üìä Example Query
SELECT
    d.year,
    d.month_name,
    p.category,
    c.customer_segment,
    s.city AS store_city,
    SUM(f.total_amount) AS total_sales
FROM fact_sales f
JOIN dim_date d ON f.date_key = d.date_key
JOIN dim_product p ON f.product_key = p.product_key
JOIN dim_customer c ON f.customer_key = c.customer_key
JOIN dim_store s ON f.store_key = s.store_key
WHERE d.year = 2024
GROUP BY d.year, d.month_name, p.category, c.customer_segment, s.city;
This query answers: "What were the total sales in 2024, broken down by month, product category, customer segment, and store city?"
