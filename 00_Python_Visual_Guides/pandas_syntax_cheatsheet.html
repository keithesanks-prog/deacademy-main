The 6 Syntax Rules that trip everyone up
1
The "Parentheses" Rule
df[df['A'] > 5 & df['B'] < 10] df[(df['A'] > 5) & (df['B'] < 10)]
Why? Python gets confused by the & symbol. You MUST wrap every single condition in parentheses ( ) so python calculates the True/False logic first.
2
The "Double Bracket" Rule
df['Col1', 'Col2'] df[['Col1', 'Col2']]
Why? One column = One string 'A'.
Multiple columns = A List of strings ['A', 'B'].
So you are asking for df[ list ], which results in double brackets [[ ]].
3
The "Assignment" Rule
df.Column = 5 df['Column'] = 5
Why? You can read with dot notation (df.Age), but avoid writing/creating columns with it. If the column doesn't exist yet, df.NewCol = 5 does nothing. Always use brackets ['NewCol'] to create.
4
The "Axis" Rule
df.drop('ColumnName') df.drop('ColumnName', axis=1)
Why? Pandas assumes you want to drop a Row (axis 0) by default. To drop a Column, you must explicitly shout axis=1 (or use axis='columns').
5
The "Inplace" Rule
df.dropna(inplace=True) df = df.dropna()
Best Practice: Most Pandas operations return a new copy of the data. They don't change the original df unless you say inplace=True. However, the modern standard is to just overwrite the variable: df = df...
6
The "Map vs Apply" Rule
df['Col'].apply({1:'A', 2:'B'}) df['Col'].map({1:'A', 2:'B'})
The Rule of Thumb:
Use .map() for simple "Swaps" (e.g. Dictionary lookup or Formatting string).
Use .apply() for "Calculations" (e.g. Math or complex Logic).
