üìù The Blueprint Decoded (Dockerfile)

File: Dockerfile
This tells the factory how to build the bus. Line by line:

  • FROM .../python:3.9
    "Start with a standard Bus Chassis from AWS that already has Python installed."
  • WORKDIR /var/task
    "Go to the main seating area."
  • COPY calendly_lambda.py .
    "Put the Team Leader (script) in the bus."
  • RUN pip install -r requirements.txt
    "Go invite all the friends listed in the guest list (requests, pandas, etc)."
  • CMD [ "calendly_lambda.lambda_handler" ]
    "When the bus starts, the Driver should ask the Team Leader (calendly_lambda) to start the 'lambda_handler' function."

ü§î Why do we need this "Bus"? (Zip vs Container)

üì¶ Old Way (Zip File)
  • Like trying to stuff a car trunk.
  • Size Limit: 250 MB (Unzipped).
  • Problem: Libraries like pandas and numpy are HEAVY. They often break the limit.
  • Result: "File too large" errors.
üöå New Way (Container Image)
  • Like a moving truck.
  • Size Limit: 10 GB! (Huge!)
  • Benefit: You can pack whatever you want (Pandas, Scikit-learn, even Chrome for scraping).
  • Result: It just works.
Action Plan: "Now that we know the mission, let's Build the Bus and Park it!"
Step 1

Reserve a Parking Spot (Create Repo)

We need a dedicated spot in the garage (ECR) for our "Calendly Bus".

aws ecr create-repository --repository-name calendly --region us-east-1
IMPORTANT: Look at the Output!
Find the line "repositoryUri". It looks like:
574181619746.dkr.ecr.us-east-1.amazonaws.com/calendly
COPY THIS! This is your specific Garage Address. You will need it for Step 4.
Step 2

Build the Bus (Docker Build)

Take the Blueprint (Dockerfile) and the Passengers (requirements.txt) and build the actual vehicle on your laptop.

docker build -t calendly .

üí° Note: If you get an error, make sure Docker Desktop is running!

Step 3

Get Garage Access (Login)

The Garage has a security gate. You need a 12-hour temporary pass.

aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 574181619746.dkr.ecr.us-east-1.amazonaws.com

‚úì Expected Output: "Login Succeeded"

Step 4

Put on License Plate (Tag)

Right now, your bus just says "Calendly". We need to stamp the full AWS address on it so it knows where to go.

docker tag [ShortName] [GarageAddress]:latest
docker tag calendly:latest 574181619746.dkr.ecr.us-east-1.amazonaws.com/calendly:latest

* Note: The long address is the one you copied from Step 1.

Step 5

Drive to the Garage (Push)

Upload the bus to the cloud.

docker push 574181619746.dkr.ecr.us-east-1.amazonaws.com/calendly:latest

Result: Your code is now safely parked in AWS! Lambda can now drive this bus whenever a scheduled event happens.

üöë Troubleshooting: "I can't find my Bus!"

If you go to Lambda and click "Browse Images" but the list is empty, don't panic.

Common Fix: The "Region" Trap

Your CLI pushed the image to N. Virginia (us-east-1).
But your AWS Console might be looking at Ohio (us-east-2) or Oregon.

üëâ Solution: Check the top-right corner of your AWS Web Console. Ensure it says "N. Virginia".
