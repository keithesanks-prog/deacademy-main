üè∑Ô∏è What is a Tag?

A tag is simply a custom Key-Value pair (a label) that you attach to an AWS resource.

  • Key: The category name (e.g., Department, Owner).
  • Value: The specific data (e.g., Finance, Alice).

Think of it like a sticky note: You stick a note that says "Property of John" on your laptop. The laptop works the same, but now everyone knows who it belongs to.

ü§î Why Do We Need Tags?

AWS accounts are chaotic. You have 100 EC2 instances. Which ones are Production? Which ones belong to Marketing? Who pays for them?

Tags are metadata (Key-Value pairs) you stick onto resources to organize them.

‚úàÔ∏è Visual Analogy: The Airport Luggage Tag

Imagine 10,000 suitcases at an airport without tags.

  • Without Tags:
    • You don't know where they go (Routing/Automation failure).
    • You don't know who owns them (Cost Allocation failure).
    • You don't know if they contain fragile items (Security/Compliance failure).
  • With Tags (Destination=LHR, Priority=High, Fragile=Yes):
    • Conveyor belts automatically route them (Resource Groups).
    • Fees are charged to the right airline (Cost Allocation).
    • Handlers treat them with care (Policy Enforcement).

üèõÔ∏è The 3 Pillars of Tagging

1. Cost Allocation (The Bill)

Goal: Charge the right team.

  • Key: CostCenter, Project, Owner
  • Value: Marketing, WebsiteRevamp, john.doe
  • Result: In AWS Cost Explorer, you filter by Project=WebsiteRevamp and see exactly $503.20 spent.

2. Automation & Operations

Goal: Do things to specific groups of resources.

  • Key: Environment, Schedule, Backup
  • Value: Production, OfficeHoursOnly, Daily
  • Result: A script runs at 6 PM: "Find all instances with Schedule=OfficeHoursOnly and Stop them." (Saves money!).

3. Access Control (ABAC - Attribute Based Access Control)

Goal: Security based on tags, not just Role names.

  • Policy: "Allow User X to start instances ONLY IF Department=User'sDepartment."
  • Result: A developer from "GreenTeam" tries to stop a "BlueTeam" server. DENIED.

üß™ Interactive Lab: Tagging with CLI

1. Tagging an Instance (Create)

# Create an instance with tags immediately
aws ec2 run-instances \
    --image-id ami-12345678 \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Environment,Value=Production},{Key=CostCenter,Value=CC123}]'

2. Filtering Resources (Find)

Find all Production servers to patch them.

aws ec2 describe-instances \
    --filters "Name=tag:Environment,Values=Production"

3. Cost Allocation (Enable)

Note: Tags don't appear in the bill automatically!

  1. Go to Billing Dashboard.
  2. Click Cost allocation tags.
  3. Select the tags you want (e.g., CostCenter).
  4. Click Activate.
  5. Wait 24 hours for them to show up in Cost Explorer.
