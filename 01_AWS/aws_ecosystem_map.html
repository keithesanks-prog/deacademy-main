"Why is this so confusing?"
Because AWS gives you Legos, not a pre-built Castle.

You are right. It IS scattered.

AWS splits everything into tiny pieces (Decoupling) so that if one piece breaks, the whole system doesn't crash. But this makes learning it feel like picking up 1,000 puzzle pieces. Let's group them into just 3 Buckets.

üóëÔ∏è The 3 Buckets of Infrastructure

Every tool you have learned falls into one of these three jobs.

1. COMPUTE

"The Brains & Muscle"

These are the things that actually run code or process logic.

EC2 Analogy: The Worker / Raw Computer

Just a remote computer. You manage everything.

ECS Analogy: The Manager / Kitchen

Runs Docker containers continuously (Long-running).

Lambda Analogy: The Uber / Task Rabbit

Runs code for 15 mins then dies (Short-running).

2. TRANSPORT

"The Roads & Mail"

These move data from A to B. They don't think; they just deliver.

API Gateway Analogy: The Front Door / Bouncer

The entry point for all users.

SQS Analogy: The Mailbox / Buffer

Holds messages until someone can read them.

Kinesis Analogy: The Conveyor Belt

Moves massive real-time data flow.

SNS Analogy: The Megaphone

Shouts one message to many people.

3. STORAGE

"The Memory"

These keep the data safe forever.

S3 Analogy: The Hard Drive / File Cabinet

Stores files (Images, CSVs, Logs).

DynamoDB Analogy: The Coat Check

Fast Key-Value store for app data.

Athena Analogy: The Librarian

Queries S3 files using SQL.
