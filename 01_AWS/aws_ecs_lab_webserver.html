Deploying: web-server-01

Rob: "Enough theory! Put on your apron. We are going to deploy a real web server called web-server-01.
We will use Fargate so we don't have to worry about buying an oven (servers)."
Step 1

The Blueprint (Task Definition)

want to use your own code?
You first need to stock the pantry! Go to Lab 00: ECR (Stocking the Pantry) to upload your image.
üåÆ Analogy: Writing the Recipe Card.

First, we need to tell AWS what "web-server-01" actually IS.

1. Go to Amazon ECS -> Task Definitions.
2. Click Create new Task Definition.
3. Family Name: web-server-01
4. Launch Type: Fargate
5. OS Architecture: Linux/X86_64
6. Task Size: 0.5 vCPU, 1 GB Memory.

Add Container Details:

1. Name: nginx-container
2. Image URI: nginx:latest (This pulls from Docker Hub)
3. Port Mappings: 80 (HTTP)
4. Click Create.
Step 2

The Kitchen (Cluster)

üèóÔ∏è Analogy: Creating the Kitchen Zone.

We need a place to run this. If you don't have a cluster, make one.

1. Go to Clusters -> Create Cluster.
2. Cluster Name: my-web-cluster
3. Infrastructure: Select AWS Fargate (Serverless).
4. Click Create.
Step 3

Order Up! (Run Task)

üç≥ Analogy: Cooking the Dish.

Now we have a Recipe (Task Def) and a Kitchen (Cluster). Let's cook! We are doing a Run Task (Ad-hoc) instead of a Service for this simple test.

1. Go into my-web-cluster.
2. Click the Tasks tab -> Run new task.
3. Launch Type: Fargate.
4. Family: Select web-server-01.
5. Networking: Choose your VPC and a Public Subnet.
6. Security Group: ENSURE Port 80 is Open to All (0.0.0.0/0).
7. Click Create.
